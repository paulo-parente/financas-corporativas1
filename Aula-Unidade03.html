1. Tipos de Relação entre Investimentos

Quando uma empresa avalia múltiplos projetos simultaneamente, a relação entre eles determina o método de análise mais adequado:

Independentes

A decisão de aceitar ou rejeitar um projeto não interfere na decisão sobre os demais. Cada projeto é avaliado isoladamente pelo critério VPL > 0.

Dependentes

A viabilidade de um projeto está condicionada à aceitação de outro. Ex.: um projeto de expansão só faz sentido se o projeto de infraestrutura for aprovado.

Mutuamente excludentes

A escolha de um projeto elimina os demais — apenas um pode ser executado. É o caso mais frequente em substituição de equipamentos ou escolha de tecnologia. Aqui surgem os três problemas típicos desta unidade.

Problemas típicos diante de múltiplos projetos: (1) restrição de capital — não há recursos para todos os projetos viáveis; (2) diferença de escala — projetos com investimentos iniciais muito distintos; (3) vida útil diferente — projetos com horizontes temporais incomparáveis diretamente.

2. Restrição de Capital

Conceito

Mesmo que múltiplos projetos apresentem VPL positivo, os recursos disponíveis para investimento podem ser limitados. Essa situação configura uma restrição de capital: nem todos os projetos viáveis podem ser executados.

Metodologia em duas etapas

Etapa 1 — Avaliar cada projeto isoladamente (VPL e TIR). Eliminar projetos com VPL < 0.
Etapa 2 — Montar carteiras de projetos que respeitem o limite orçamentário. Com \(n\) projetos viáveis, existem \(2^n - 1\) carteiras possíveis. Selecionar a de maior VPL total.

Atenção — Análise Incremental para excludentes

Quando os projetos são mutuamente excludentes, o maior VPL não é suficiente — é preciso verificar se o incremento de investimento entre alternativas se justifica. Calcula-se o fluxo de caixa incremental e sua TIR: se TIRincremental > TMA, o investimento adicional cria valor.

Análise Incremental para projetos mutuamente excludentes

Quando os projetos são mutuamente excludentes — só um pode ser escolhido — a comparação direta pelo VPL pode ser enganosa se os investimentos iniciais forem muito diferentes. O procedimento correto é a análise incremental.

\[ \text{Fluxo incremental} = FC_{\text{maior}} - FC_{\text{menor}} \] \[ \text{Se } TIR_{\text{incremental}} > TMA \Rightarrow \text{escolher o projeto de maior investimento} \]

3. Diferentes Níveis de Escala

O Problema de Escala

Quando dois projetos mutuamente excludentes têm investimentos iniciais muito diferentes, a TIR e o VPL podem apontar decisões opostas. O VPL mede riqueza absoluta; a TIR mede retorno relativo. Com escalas distintas, um projeto pode ter TIR maior mas criar menos valor absoluto.

Fluxo de Caixa Incremental

A solução é decompor o projeto maior em dois componentes:

\[ B = A + (B - A) \] \[ \text{Calcular a TIR do fluxo } (B - A) \text{ e comparar com a TMA} \]

Se \(TIR_{B-A} > TMA\): o investimento incremental cria valor → escolher B.
Se \(TIR_{B-A} < TMA\): o investimento incremental destrói valor → escolher A.

Taxa de Fischer

A Taxa de Fischer é a TMA para a qual \(\text{VPL}_A = \text{VPL}_B\). Abaixo dessa taxa, o projeto de maior escala é melhor; acima dela, o de menor escala pode ser preferível. Coincide com a TIR do fluxo incremental \((B - A)\).

4. Projetos com Vida Útil Distintos

O Problema do Horizonte Temporal

Quando projetos têm vidas úteis diferentes, o VPL não é diretamente comparável: um projeto de 8 anos "oferece" mais períodos de benefício do que um de 4 anos. A forma de tratar esse problema depende da intenção de continuidade da atividade.

Caso 1 — Descontinuidade

Os projetos serão encerrados ao término de suas vidas úteis. Os objetivos foram alcançados e não há necessidade de reposição. Nesse caso, a comparação direta do VPL é válida, mesmo com vidas distintas.

Caso 2 — Continuidade

A atividade continuará após o término do projeto. O equipamento precisará ser reposto. A comparação direta do VPL é inadequada — os benefícios são medidos em horizontes distintos.

Solução 1 — Repetição pelo MMC

Repetir os fluxos de caixa de cada projeto até o Mínimo Múltiplo Comum (MMC) das vidas úteis, criando um horizonte idêntico. Limitação: se os projetos tiverem 5, 6 e 7 anos, o MMC é 210 anos — inviável na prática.

Solução 2 — Valor Uniforme Equivalente (VUE)

Converte o VPL em uma anuidade anual equivalente, permitindo comparar projetos de qualquer vida útil sem repetição:

\[ VUE = \frac{VPL}{(P/A;\,i;\,n)} = VPL \times (A/P;\,i;\,n) \] onde \quad (A/P;\,i;\,n) = \frac{i(1+i)^n}{(1+i)^n - 1}

O projeto com maior VUE é o preferível em situações de continuidade. O método de repetição e o VUE conduzem à mesma decisão.
